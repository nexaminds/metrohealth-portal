How NexAI handles your data

This portal sits behind the NexAI platform — autonomous AI agents that monitor production, identify problems within seconds, and propose fixes through standard human-reviewed pull requests. Here's what you should know:

PHI never leaves our infrastructure unhashed

Patient identifiers — names, emails, MRN, DOB, SSN — are scrubbed at the application boundary using HMAC-SHA256 with tenant-scoped salts. Traces, audit logs, and observability data store only the hash format hmac_sha256:v1:<salt>:<16-hex> — never the raw value. This is verified per-trace by automated evaluators.

No AI action mutates production without a human

Every fix the AI proposes is opened as a standard GitHub pull request. CI runs the regression test the AI itself wrote. A human engineer must review and merge before anything ships. This is enforced at the platform level — not policy, code.

Every AI action is auditable

Each incident generates an immutable JSON record conforming to a published schema (prod_mutation_audit). The record names every API call, every credential issuance, every state read, and every agent that touched the work. Procurement and compliance can trace any action back to its root cause in seconds.

PHI in this demo is synthetic

All names, phone numbers, dates of birth, and identifiers shown here are generated from Synthea — an open-source synthetic patient generator. No real patient data is present in this demo environment.
